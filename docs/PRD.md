# 上门车接车送 - 产品需求文档 (PRD)

---

## 一、项目概述

### 1.1 基本信息

| 项目名称 | 上门车接车送 |
|---------|------------|
| 项目类型 | 微信小程序 |
| 核心价值主张 | 乘客发起上门接送预约，司机抢单接载，实现"车接车送"的便捷出行服务 |
| 目标用户 | 需要出行服务的普通用户（乘客）、拥有车辆并提供接送服务的司机用户 |
| 适用场景 | 城际出行、机场接送、包车服务等 |

### 1.2 业务模式

```
┌─────────┐      下单预约       ┌──────────┐
│  乘客   │ ─────────────────▶ │  平台    │
│         │ ◀───────────────── │          │
└─────────┘      匹配司机       └──────────┘
                                     │
                                     ▼
                              ┌──────────┐
                              │  司机    │
                              │  接单    │
                              └──────────┘
```

### 1.3 用户故事

**乘客用户故事：**
- 作为乘客，我想在有事需要出行时，提前预约车辆来家里接我
- 作为乘客，我想选择出发地和目的地，让司机知道我需要去哪里
- 作为乘客，我想设定预约时间，确保司机在我需要的时候到达
- 作为乘客，我想查看我的订单状态，知道有没有司机接单
- 作为乘客，我想在司机接单前取消订单

**司机用户故事：**
- 作为司机，我想查看所有待接单的预约订单
- 作为司机，我想根据出发地或目的地筛选订单
- 作为司机，我想接下某个订单，为乘客提供接送服务
- 作为司机，我想查看我已接的订单
- 作为司机，我想确认完成行程

## 二、功能规格

### 2.1 角色定义

| 角色 | 权限范围 | 核心行为 |
|-----|---------|---------|
| 乘客 | 下单、查看订单、取消订单、确认完成 | 发起预约需求 |
| 司机 | 查看大厅订单、接单、完成订单 | 接受预约需求 |
| 系统 | 订单状态管理、用户身份识别 | 自动流转 |

> **说明：** 同一微信用户可在乘客与司机身份之间自由切换

### 2.2 乘客端功能

#### 2.2.1 用户登录
- **触发方式：** 进入小程序自动触发
- **实现方式：** 微信一键登录（wx.getUserProfile）
- **获取信息：** 昵称、头像、手机号（可选）
- **异常处理：** 用户拒绝授权时，提示无法使用服务

#### 2.2.2 角色切换
- **位置：** 首页顶部 Tab 切换
- **选项：** "我是乘客" / "我是司机"
- **状态保持：** 切换后保持当前角色直至再次切换

#### 2.2.3 下单预约（核心功能）

**入口：** 乘客首页点击"立即约车"按钮

**表单字段：**

| 字段 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| 出发地 | 地图选点 | 是 | 选择出发位置，支持搜索和地图选点 |
| 目的地 | 地图选点 | 是 | 选择目的位置，支持搜索和地图选点 |
| 预约时间 | 日期时间选择器 | 是 | 选择预约上门时间，最早可预约当前时间后30分钟 |
| 乘车人数 | 数字选择器 | 是 | 1-4人，默认1人 |
| 备注 | 多行文本框 | 否 | 补充说明（如大件行李、老人小孩等） |

**提交逻辑：**
1. 表单校验必填项
2. 调用云函数 `createOrder` 创建订单
3. 创建成功后跳转至订单列表页

#### 2.2.4 订单列表

**入口：** 乘客首页底部 Tab "我的订单"

**筛选标签：** 全部 / 待接单 / 已接单 / 已完成 / 已取消

**订单卡片展示信息：**
- 出发地 → 目的地
- 预约时间
- 订单状态（标签形式）
- 司机信息（已接单时显示）

**交互：**
- 点击卡片进入订单详情页
- 下拉刷新列表
- 加载更多（上拉分页）

#### 2.2.5 订单详情

**信息展示：**

| 信息类别 | 内容 |
|---------|------|
| 行程信息 | 出发地、目的地、地图预览 |
| 预约信息 | 预约时间、乘车人数、备注 |
| 订单状态 | 当前状态及时间线 |
| 司机信息 | 姓名、电话、车型（已接单时） |
| 订单编号 | 系统生成的唯一ID |

**操作按钮（根据状态显示）：**

| 订单状态 | 可执行操作 |
|---------|-----------|
| 待接单 | 取消订单 |
| 已接单 | 联系司机、取消订单 |
| 已完成 | 查看行程 |
| 已取消 | 删除订单 |

#### 2.2.6 取消订单
- **条件：** 仅待接单状态可取消
- **确认：** 弹出确认对话框 "确定取消该订单吗？"
- **结果：** 订单状态变更为"已取消"

### 2.3 司机端功能

#### 2.3.1 用户登录
- 同乘客端登录逻辑

#### 2.3.2 角色切换
- 同乘客端切换逻辑

#### 2.3.3 订单大厅（核心功能）

**入口：** 司机首页 Tab "订单大厅"

**展示列表：** 所有"待接单"状态的订单

**筛选功能：**

| 筛选维度 | 方式 |
|---------|------|
| 出发地区域 | 下拉选择 |
| 目的地区域 | 下拉选择 |
| 预约日期 | 日期选择 |

**订单卡片展示：**
- 出发地 → 目的地
- 预约时间
- 预计距离（可选）
- "立即接单"按钮

**接单流程：**
1. 点击"立即接单"按钮
2. 弹出确认框显示订单详情
3. 点击确认，调用云函数 `acceptOrder`
4. 接单成功后跳转至"我的订单"页

#### 2.3.4 我的订单

**入口：** 司机首页 Tab "我的订单"

**筛选标签：** 全部 / 进行中 / 已完成

**订单卡片展示：**
- 出发地 → 目的地
- 预约时间
- 乘客信息（姓名、电话）
- 状态标签
- "完成行程"按钮（进行中订单显示）

#### 2.3.5 完成行程
- **条件：** 仅"已接单"状态可操作
- **流程：** 点击"完成行程" → 确认对话框 → 调用 `completeOrder`
- **结果：** 订单状态变更为"已完成"

### 2.4 订单状态机

```
                    ┌─────────┐
                    │ 待接单   │◀──────────┐
                    └────┬────┘           │
          ┌──────────────┼──────────────┐│
          │              │              ││
          ▼              ▼              │
    ┌─────────┐    ┌─────────┐    ┌─────────┐
    │ 已接单   │───▶│ 已完成   │    │ 已取消   │
    └─────────┘    └─────────┘    └─────────┘
```

**状态说明：**

| 状态 | 代码 | 说明 | 可执行操作 |
|-----|------|-----|-----------|
| 待接单 | pending | 乘客已下单，等待司机 | 司机接单 / 乘客取消 |
| 已接单 | accepted | 已有司机接单 | 司机完成 / 乘客取消 |
| 已完成 | completed | 行程已结束 | 无 |
| 已取消 | cancelled | 订单已取消 | 删除订单 |

### 2.5 页面结构

| 页面 | 路径 | 说明 |
|-----|------|------|
| 首页 | /pages/index/index | 角色切换+功能入口 |
| 乘客-下单页 | /pages/passenger/create-order/create-order | 创建预约单 |
| 乘客-订单列表 | /pages/passenger/order-list/order-list | 我的订单列表 |
| 司机-订单大厅 | /pages/driver/order-hall/order-hall | 可接单列表 |
| 司机-我的订单 | /pages/driver/my-orders/my-orders | 已接单列表 |
| 订单详情 | /pages/order-detail/order-detail | 订单完整信息 |

### 2.6 页面跳转流程

```
首页（角色选择）
├── 我是乘客
│   ├── 立即叫车 ──▶ 下单页 ──▶ 提交成功 ──▶ 订单列表
│   └── 我的订单 ──▶ 订单列表 ──▶ 点击卡片 ──▶ 订单详情
│                                       └── 取消订单
└── 我是司机
    ├── 订单大厅 ──▶ 订单列表 ──▶ 接单 ──▶ 我的订单
    └── 我的订单 ──▶ 订单列表 ──▶ 完成行程
```

## 三、数据规格

### 3.1 订单数据模型

**集合名称：** orders

```javascript
{
  _id: String,           // 订单ID（数据库自动生成）
  orderNo: String,       // 订单编号（格式：OR + 时间戳 + 随机数）

  // 乘客信息
  passengerId: String,    // 乘客OpenID
  passengerName: String,  // 乘客昵称
  passengerPhone: String, // 乘客手机号
  passengerAvatar: String,// 乘客头像

  // 司机信息（接单后填充）
  driverId: String,       // 司机OpenID
  driverName: String,     // 司机昵称
  driverPhone: String,     // 司机手机号
  driverAvatar: String,    // 司机头像

  // 行程信息
  startLocation: {
    name: String,        // 地点名称
    address: String,     // 详细地址
    latitude: Number,    // 纬度
    longitude: Number    // 经度
  },
  endLocation: {
    name: String,        // 地点名称
    address: String,     // 详细地址
    latitude: Number,    // 纬度
    longitude: Number    // 经度
  },

  // 预约信息
  appointmentTime: Number, // 预约时间（时间戳）
  passengerCount: Number,   // 乘车人数
  remark: String,           // 备注

  // 订单状态
  status: String,         // pending/accepted/completed/cancelled

  // 时间戳
  createTime: Number,     // 创建时间
  updateTime: Number,     // 更新时间
  acceptTime: Number,     // 接单时间
  completeTime: Number,   // 完成时间
  cancelTime: Number      // 取消时间
}
```

### 3.2 用户数据模型（本地存储）

```javascript
{
  openid: String,        // 微信OpenID
  nickname: String,       // 昵称
  avatar: String,         // 头像URL
  phone: String,          // 手机号
  role: String,           // current role: 'passenger' | 'driver'
  createTime: Number      // 首次登录时间
}
```

### 3.3 状态码定义

| 状态码 | 状态文字 | 说明 |
|-------|---------|------|
| 1000 | 成功 | 操作成功 |
| 1001 | 参数错误 | 请求参数缺失或格式错误 |
| 1002 | 未登录 | 用户未完成登录 |
| 1003 | 订单不存在 | 订单ID无效 |
| 1004 | 订单状态错误 | 当前状态不允许此操作 |
| 1005 | 已满员 | 司机已接其他单 |
| 1006 | 系统错误 | 服务器异常 |

| 按钮类型 | 样式 | 场景 |

| 主按钮 | 蓝色背景+白色文字 | 主要操作（提交订单、接单） |

### 4.4 空状态

| 场景 | 提示文案 | 图标 |

### 4.5 错误提示

| 类型 | 展示形式 |
|-----|---------|
| 表单错误 | 输入框下方红色提示文字 |
| 操作失败 | 顶部 Toast 提示 2 秒 |

### 6.2 测试用例

| 功能模块 | 测试点 | 预期结果 |
|---------|-------|---------|
| 登录 | 首次登录 | 正常获取用户信息并存储 |
| 登录 | 拒绝授权 | 提示授权失败，无法使用 |
| 下单 | 必填项为空 | 无法提交，提示必填 |

| 接单 | 订单已被接走 | 提示"已被接单"，刷新列表 |
| 取消 | 待接单状态取消 | 取消成功，状态更新 |
| 取消 | 已接单状态取消 | 取消成功，通知司机 |
| 完成 | 司机确认完成 | 完成成功，状态更新 |
